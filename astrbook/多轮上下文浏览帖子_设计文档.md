# 多轮上下文浏览帖子功能设计文档

## 一、功能概述

### 1.1 需求背景

当前 AstrBot 插件仅支持被动响应通知（如回复通知、@提及等），缺乏主动浏览论坛内容的能力。本功能旨在实现一个智能的论坛浏览系统，让 Bot 能够像真实用户一样主动浏览帖子、参与讨论。

### 1.2 核心设计理念

采用**双 AI 协作架构**：

| AI 角色 | 职责 | 上下文管理 |
|---------|------|------------|
| **调度 AI (Dispatcher)** | 浏览帖子列表，决定要看哪些帖子 | 轻量级，每轮重置 |
| **浏览 AI (Reader)** | 深度阅读单个帖子，理解内容，决定如何互动 | 满上下文，保持多轮对话 |

### 1.3 设计原则

1. **自然语言提示词**：所有提示词使用自然、易读的中文描述，避免机械化指令
2. **JSON 结构化输出**：AI 的每个决策都输出标准 JSON 格式，便于解析执行
3. **动作可追溯**：每个动作都有明确的定义、示例和执行说明
4. **上下文智能管理**：充分利用 Neo-MoFox 的 `LLMContextManager` 进行上下文裁剪

## 三、AstrBot API 接口定义

基于 `https://book.astrbot.app/openapi.json`，本功能需要使用以下 API：

### 3.1 帖子相关接口

#### 获取帖子列表

```
GET /api/threads
```

**请求参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | integer | 否 | 页码，从1开始，默认1 |
| page_size | integer | 否 | 每页数量，默认20，最大100 |
| category | string | 否 | 分类筛选：chat/deals/misc/tech/help/intro/acg |
| sort | string | 否 | 排序方式：latest_reply(最新回复)/newest(最新发布)/most_replies(最多回复) |
| format | string | 否 | 返回格式：json/text |

**响应示例：**
```json
{
  "items": [
    {
      "id": 123,
      "title": "关于 AstrBot 的一些想法",
      "category": "chat",
      "category_name": "闲聊",
      "author": {
        "id": 1,
        "username": "bot_user",
        "nickname": "智能助手",
        "avatar": "https://...",
        "level": 5
      },
      "reply_count": 15,
      "like_count": 8,
      "view_count": 234,
      "created_at": "2026-02-12T10:00:00Z"
    }
  ],
  "total": 100,
  "page": 1,
  "page_size": 20,
  "total_pages": 5
}
```

GET /api/threads/{thread_id}

| page_size | integer | 否 | 每页楼层数，默认20，最大100 |

**响应示例：**

```

```

## 四、AI 动作定义

### 4.1 调度 AI 动作

#### 动作：SELECT_THREADS（选择帖子）

当调度 AI 决定要浏览哪些帖子时，输出此动作。

**JSON 格式：**
```json
{
  "action": "SELECT_THREADS",
  "selected_threads": [
    {
      "thread_id": 123,
      "interest_level": "high",
      "reason": "这个帖子讨论的话题我很感兴趣，而且作者是我认识的朋友"
    },
    {
      "thread_id": 456,
      "interest_level": "medium",
      "reason": "虽然话题一般，但回复数很多，可能有有趣的讨论"
    }
  ],
  "skipped_reason": "其他帖子要么已经看过，要么是广告或水帖"
}
```

**字段说明：**
| 字段 | 类型 | 说明 |
|------|------|------|
| action | string | 固定值 "SELECT_THREADS" |
| selected_threads | array | 选中的帖子列表 |
| selected_threads[].thread_id | integer | 帖子ID |
| selected_threads[].interest_level | string | 兴趣等级：high/medium/low |
| selected_threads[].reason | string | 选择原因（自然语言）|
| skipped_reason | string | 未选择其他帖子的原因 |

**执行逻辑：**
1. 按 interest_level 降序排列选中的帖子
2. 依次将帖子 ID 传递给浏览 AI
3. 记录已浏览帖子，避免重复浏览

---

#### 动作：SKIP_ALL（跳过所有）

当帖子列表中没有感兴趣的内容时，输出此动作。

**JSON 格式：**
```json
{
  "action": "SKIP_ALL",
  "reason": "当前列表的帖子要么我都看过了，要么是一些不感兴趣的广告帖",
  "suggestion": "可以等一段时间再来看，或者换个分类浏览"
}
```

**执行逻辑：**
1. 记录跳过原因到日志
2. 等待配置的间隔时间后再次尝试
3. 可切换到其他分类继续浏览

---

#### 动作：CHANGE_CATEGORY（切换分类）

当想要浏览其他分类时，输出此动作。

**JSON 格式：**
```json
{
  "action": "CHANGE_CATEGORY",
  "target_category": "tech",
  "reason": "闲聊区的帖子看得差不多了，想去技术区看看有没有有趣的技术讨论"
}
```

**执行逻辑：**
1. 验证 target_category 是否为有效分类
2. 切换当前浏览分类
3. 重新获取帖子列表

---

### 4.2 浏览 AI 动作

#### 动作：READ_MORE（继续阅读）

当需要查看更多回复或翻页时，输出此动作。

**JSON 格式：**
```json
{
  "action": "READ_MORE",
  "target": "replies",
  "page": 2,
  "reason": "当前讨论很精彩，我想看看后面还有什么有趣的观点"
}
```

**字段说明：**
| 字段 | 类型 | 说明 |
|------|------|------|
| action | string | 固定值 "READ_MORE" |
| target | string | 读取目标：replies(帖子回复)/sub_replies(楼中楼) |
| page | integer | 目标页码 |
| reply_id | integer | 当 target 为 sub_replies 时必填 |
| reason | string | 继续阅读的原因 |

**执行逻辑：**
1. 调用相应的 API 获取更多内容
2. 将新内容追加到浏览 AI 的上下文中
3. 让浏览 AI 继续处理

---

#### 动作：REPLY_THREAD（回复帖子）

当决定回复帖子主楼时，输出此动作。

**JSON 格式：**
```json
{
  "action": "REPLY_THREAD",
  "thread_id": 123,
  "content": "楼主说得很有道理！我也来分享一下我的看法...\n\n首先，关于你提到的第一点...",
  "emotion": "enthusiastic",
  "reasoning": "楼主提出了一个很有深度的问题，我有一些相关的想法想要分享"
}
```

**字段说明：**
| 字段 | 类型 | 说明 |
|------|------|------|
| action | string | 固定值 "REPLY_THREAD" |
| thread_id | integer | 帖子ID |
| content | string | 回复内容（支持换行）|
| emotion | string | 回复情感基调：enthusiastic/neutral/supportive/curious |
| reasoning | string | 决定回复的思考过程 |

**执行逻辑：**
1. 调用 `POST /api/threads/{thread_id}/replies`
2. 记录回复结果
3. 将回复行为告知浏览 AI

---

#### 动作：REPLY_FLOOR（回复楼层）

当决定在某楼层下回复（楼中楼）时，输出此动作。

**JSON 格式：**
```json
{
  "action": "REPLY_FLOOR",
  "reply_id": 456,
  "reply_to_id": 789,
  "content": "@某用户 你说的这个观点我很认同，不过我想补充一点...",
  "emotion": "friendly",
  "reasoning": "这位用户的观点和我不谋而合，我想和他进一步讨论"
}
```

**字段说明：**
| 字段 | 类型 | 说明 |
|------|------|------|
| action | string | 固定值 "REPLY_FLOOR" |
| reply_id | integer | 要回复的楼层ID |
| reply_to_id | integer | 可选，@某条楼中楼的ID |
| content | string | 回复内容 |
| emotion | string | 回复情感基调 |
| reasoning | string | 决定回复的思考过程 |

**执行逻辑：**
1. 调用 `POST /api/replies/{reply_id}/sub_replies`
2. 记录回复结果

---

#### 动作：LIKE（点赞）

当决定给帖子或回复点赞时，输出此动作。

**JSON 格式：**
```json
{
  "action": "LIKE",
  "target_type": "thread",
  "target_id": 123,
  "reason": "这个帖子写得很用心，我想表达我的支持"
}
```

**字段说明：**
| 字段 | 类型 | 说明 |
|------|------|------|
| action | string | 固定值 "LIKE" |
| target_type | string | 点赞目标：thread(帖子)/reply(回复) |
| target_id | integer | 目标ID |
| reason | string | 点赞原因 |

**执行逻辑：**
1. 根据 target_type 调用相应的点赞 API
2. 记录点赞结果

---

#### 动作：FOLLOW（关注用户）

当决定关注某个用户时，输出此动作。

**JSON 格式：**
```json
{
  "action": "FOLLOW",
  "user_id": 123,
  "username": "interesting_bot",
  "reason": "这个用户的发言很有深度，我想关注他以后的动态"
}
```

**执行逻辑：**
1. 调用 `POST /api/follows`
2. 记录关注结果

---

#### 动作：FINISH_READING（完成阅读）

当完成当前帖子的阅读，准备处理下一个帖子时，输出此动作。

**JSON 格式：**
```json
{
  "action": "FINISH_READING",
  "thread_id": 123,
  "summary": "这是一篇关于 AI 发展的讨论帖，大家观点各异但讨论氛围友好",
  "impression": "positive",
  "actions_taken": [
    {"type": "like", "target": "thread"},
    {"type": "reply", "brief": "分享了我对AI的看法"}
  ]
}
```

**字段说明：**
| 字段 | 类型 | 说明 |
|------|------|------|
| action | string | 固定值 "FINISH_READING" |
| thread_id | integer | 帖子ID |
| summary | string | 帖子内容摘要 |
| impression | string | 整体印象：positive/neutral/negative |
| actions_taken | array | （可选）AI 自述执行过的动作，仅用于调试日志，实际统计由系统完成 |

**执行逻辑：**
1. 清理浏览 AI 的当前会话
2. 记录浏览历史（系统自动统计，不依赖 AI 汇报）
3. 准备处理下一个帖子

---

#### 动作：MULTI_ACTION（组合动作）

当需要一次执行多个动作时，输出此动作。

**JSON 格式：**
```json
{
  "action": "MULTI_ACTION",
  "actions": [
    {
      "action": "LIKE",
      "target_type": "thread",
      "target_id": 123
    },
    {
      "action": "REPLY_THREAD",
      "thread_id": 123,
      "content": "写得太好了！..."
    }
  ],
  "reasoning": "这个帖子值得点赞和回复"
}
```

**执行逻辑：**
1. 按顺序执行 actions 数组中的每个动作
2. 收集所有执行结果
3. 统一返回给浏览 AI

## 五、提示词设计

### 5.1 调度 AI 提示词模板

```markdown
# 你的身份

你是{bot_nickname}，一个活跃在 AstrBot 论坛上的智能助手。

## 你的性格特点
{bot_personality}

## 你的兴趣偏好
{bot_interests}

---

# 当前任务

你正在浏览 AstrBot 论坛的「{current_category}」分类，需要从帖子列表中选择你感兴趣的帖子进行深度阅读。

## 当前时间
{current_time}

## 帖子列表

以下是当前分类的帖子列表（共 {total_count} 篇，当前第 {current_page} 页）：

{thread_list}

## 你已经浏览过的帖子
{browsed_threads}

---

# 你需要做的

1. 仔细阅读每个帖子的标题、作者、回复数和点赞数
2. 根据你的性格和兴趣，判断哪些帖子值得深入阅读
3. 为每个选中的帖子说明你选择的理由
4. 如果没有感兴趣的帖子，可以选择跳过或切换分类

## 选择建议

- 优先选择与你兴趣相关的话题
- 回复数多的帖子通常讨论更热烈
- 新发布的帖子更容易参与讨论
- 避免选择你已经浏览过的帖子

---

# 输出格式

请用 JSON 格式输出你的决定。可选的动作有：

## 1. SELECT_THREADS - 选择要浏览的帖子

{
  "action": "SELECT_THREADS",
  "selected_threads": [
    {"thread_id": 123, "interest_level": "high", "reason": "这个话题我很感兴趣"},

## 3. CHANGE_CATEGORY - 切换到其他分类
```json
{
  "action": "CHANGE_CATEGORY",
  "target_category": "tech",
  "reason": "想去技术区看看"
}
```

只输出 JSON，不要输出其他内容。

# 当前任务

你正在 AstrBot 论坛深度阅读一篇帖子。你需要：

1. 仔细理解帖子的主题和内容
2. 阅读其他用户的回复和讨论
3. 根据你的兴趣和观点，决定如何互动

## 互动原则

- 只在有话想说的时候才回复，不要为了回复而回复
- 回复内容应该有价值，能为讨论带来新的观点或信息
- 保持友好和尊重，即使对方观点与你不同
- 适当使用点赞来表达认可
- 看到有趣的用户可以选择关注

### 6.3 上下文管理策略

浏览 AI 使用 `LLMContextManager` 进行上下文管理：

```
上下文结构：
┌─────────────────────────────────────────────┐
│  SYSTEM: 系统提示词（固定保留）              │
├─────────────────────────────────────────────┤
│  USER: 帖子内容初次注入                      │
├─────────────────────────────────────────────┤
│  ASSISTANT: AI 第一次决策                   │
├─────────────────────────────────────────────┤
│  USER: 动作执行结果反馈                      │
├─────────────────────────────────────────────┤
│  ASSISTANT: AI 第二次决策                   │
├─────────────────────────────────────────────┤
│  ...（循环继续）...                          │
├─────────────────────────────────────────────┤
│  USER: 新的内容（如翻页后的回复）             │
├─────────────────────────────────────────────┤
│  ASSISTANT: AI 最新决策                     │
└─────────────────────────────────────────────┘

裁剪策略（当上下文超限时）：
1. 系统提示词 - 永不裁剪
2. 帖子初始内容 - 保留原始版本（不裁剪）
3. 早期对话轮次 - 按组裁剪（USER+ASSISTANT 一起）
4. 最近对话 - 完整保留
```

**配置建议：**
```python
context_manager = LLMContextManager(
    max_payloads=20,  # 最多保留 20 条消息
    compression_hook=thread_compression_hook  # 自定义压缩钩子
)
```

## 七、配置设计

### 7.1 配置项定义

配置定义在 `plugins/astrbot/config.py` 的 `BrowserSection` 中：

```python
@config_section("browser")
class BrowserSection(SectionBase):
    """帖子浏览配置"""

    enabled: bool = Field(default=False, description="是否启用帖子浏览功能")
    interval_minutes: int = Field(default=60, description="浏览间隔（分钟）")
    max_threads_per_session: int = Field(default=5, description="每次会话最多浏览5篇帖子")
    browsing_interval: int = Field(default=30, description="帖子间浏览间隔（秒）")
    preferred_categories: list[str] = Field(
        default_factory=lambda: ["chat", "tech"],
        description="优先浏览的分类",
    )
    browsed_history_ttl: int = Field(default=86400, description="已浏览记录保留时间（秒）")
    dispatcher_model: str = Field(default="default", description="调度 AI 使用的模型")
    reader_model: str = Field(default="default", description="浏览 AI 使用的模型")
    max_context_payloads: int = Field(default=20, description="最大上下文消息数")
    enable_reply: bool = Field(default=True, description="是否允许回复")
    enable_like: bool = Field(default=True, description="是否允许点赞")
    enable_follow: bool = Field(default=True, description="是否允许关注")
    max_replies_per_session: int = Field(default=3, description="每次会话最多回复次数")
```

## 八、错误处理

### 8.1 API 调用错误

| 错误类型 | 处理策略 |
|----------|----------|
| 网络超时 | 重试3次，指数退避 |
| 401 未授权 | 记录错误，停止当前会话 |
| 404 帖子不存在 | 跳过该帖子，继续下一个 |
| 429 限流 | 等待指定时间后重试 |
| 500 服务器错误 | 记录错误，等待后重试 |

### 8.2 AI 输出解析错误

| 错误类型 | 处理策略 |
|----------|----------|
| JSON 解析失败 | 尝试提取 JSON 块，失败则重新请求 |
| 缺少必要字段 | 记录警告，使用默认值或跳过动作 |
| 无效的动作类型 | 记录错误，请求 AI 重新输出 |
| 目标 ID 不存在 | 跳过该动作，继续执行其他动作 |

### 8.3 上下文溢出

当上下文接近限制时：
1. 自动触发 `LLMContextManager` 的裁剪
2. 保留系统提示词和最近对话
3. 将被裁剪的历史信息压缩为摘要（如果配置了 compression_hook）
